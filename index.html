No-LDA GB1 workflow

GB1 No-LDA Co-Optimization Demo

Upload the CSV inputs from the package. The page scores single mutations with ESM/LM plausibility, selects Round-1 candidates, validates them against measured binding and stability values, then recombines selected singles into Round-2 double mutants. No LDA model or graph 3 is used.

1. ScoreUse uploaded ESM `lm_score` values, or a fallback proxy if ESM scores are missing.
2. SelectPick the top Round-1 single mutants by LM plausibility.
3. ValidatePlot selected mutations using measured binding and stability data.
4. RecombineCreate double mutants from selected singles and check measured binding.

Inputs

Use `gb1_fitness_singles.csv`. Provides measured binding fitness. Use `gb1_stability_deltaG.csv`. Provides the source stability score. Use `gb1_fitness_doubles.csv`. Used only for Round-2 validation. Use `gb1_lm_scores.csv` for the current valid ESM2-backed run. How many top LM-plausible single mutants to carry into Round 2.
Waiting for CSV files.
Recommended: upload `gb1_lm_scores.csv` so the page uses the current ESM2-backed `lm_score` values.
Stability note: this demo uses the source-reported GB1 stability scale from the package. It does not mean generic folding reaction deltaG where lower/more negative is favorable.
