No-LDA GB1 workflow

GB1 Multi-Mutant Co-Optimization App

Run with the packaged GB1 CSV inputs, or upload replacement CSVs. The page scores single mutations with ESM/LM plausibility, selects Round-1 candidates, validates them against measured binding and stability values, then recombines selected singles into double and higher-order multi-mutant candidates. No LDA model or graph 3 is used.

1. ScoreUse uploaded ESM `lm_score` values, or a fallback proxy if ESM scores are missing.
2. SelectPick the top Round-1 single mutants by LM plausibility.
3. ValidatePlot selected mutations using measured binding and stability data.
4. RecombineCreate double mutants from selected singles and check measured binding.

Inputs

Default: `data/gb1_fitness_singles.csv`. Upload here only if replacing the packaged data. Default: `data/gb1_stability_deltaG.csv`. Provides the source stability score. Default: `data/gb1_fitness_doubles.csv`. Used only for Round-2 validation. Default: `data/gb1_lm_scores.csv` for the current valid ESM2-backed run. How many top LM-plausible single mutants to carry into Round 2. Use 2 for doubles, 3 for triples, or 4 for quadruples. Higher orders are capped to avoid lag. Limits generated higher-order combinations so the browser stays responsive.
Ready. Click Run to use the packaged GB1 CSV files.
Recommended: open this app with `start_app.bat`, click Run on this page, then open the Outputs / Results page when the run is finished.
Stability note: this demo uses the source-reported GB1 stability scale from the package. It does not mean generic folding reaction deltaG where lower/more negative is favorable.

How This App Uses The Inputs

The input tab is where you choose the measured datasets and set how far beyond double mutants the app should explore.

Guide
Measured dataBinding and stability CSVs anchor the graphs and validation tables.
LM scoreESM/LM plausibility ranks which singles enter recombination.
Multi-mutantsTriples/quads are ranked by additive stability, additive LM score, and single-mutant binding evidence.
Safety capThe max multi-mutant limit prevents huge combination counts from slowing the page.
